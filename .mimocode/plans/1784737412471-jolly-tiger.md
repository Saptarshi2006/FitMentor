# Auth: Single-cookie architecture

## Target Architecture

**One cookie in browser** → one KV entry with everything:
```
Cookie: fitmentor_session=sess_abc123
  ↓
KV: sess_abc123 → { sub, email, name, provider, rememberToken: "rem_xyz789", createdAt }
  ↓ (if session expired/missing — try renew)
KV: rem_xyz789 → { sub, email, name, provider }
  ↓ (if valid)
Create new session entry, update cookie
```

No `fitmentor_remember` cookie. Remember token lives inside the session KV entry.

2. Session expires, return → `checkSession` calls `renewSession()`, new session created from remember entry
